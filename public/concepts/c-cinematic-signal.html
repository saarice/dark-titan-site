03 · Method

From intent to production.

Three moves. The first two are yours; the third runs without you.

01 — DEFINE

Define the outcome

Describe what to build or modernize in a plain TASK.md.

$ vim TASK.md
02 — CONFIGURE

Configure the pipeline

Set the flow: stages, prompts, gates, loops, escalation.

$ darktitan import flow implement.yaml
03 — RUN

Run the factory

Launch and watch it ship production-grade code.

$ darktitan run implement.yaml my-app
04 · Capabilities

Built to ship software that ships.

/ 01

Multi-Stage Pipelines

Chain agents into named stages — plan, implement, test, review, deploy — each with its own prompt.

/ 02

Quality Gates

AI-evaluated, human-approved, or test-passing checkpoints decide whether work advances, loops, or escalates.

/ 03

Full Observability

Every prompt, change, gate decision, and deploy is logged. A complete audit trail by default.

/ 04

Pause & Resume

Stop a run mid-flight, inspect, intervene, and pick up exactly where it left off.

/ 05

Legacy Modernization

Point the factory at existing code. It reads patterns, refactors, and migrates with the same rigor.

/ 06

Greenfield Delivery

From a blank repo to a deployed, tested service — defined once, delivered end to end.

05 · Under the hood

It runs in your codebase, under your control.

DarkTitan spawns Claude Code agents as subprocesses, right where your code lives. One CLI drives the whole lifecycle.

  • init / new / import / run — stand up a project and launch a flow.
  • status & logs <flow-id> — see exactly what every agent did.
  • pause · resume · stop — take the wheel at any moment.
  • ui — open the live console to watch the factory work.
$ darktitan init
✓ workspace ready
$ darktitan new project my-app
✓ project · ./my-app
$ darktitan import flow implement.yaml my-app
✓ flow registered · implement-and-review
$ darktitan run implement.yaml my-app
▸ implement … done
▸ review (ai-evaluation) … pass
⏸ escalation: pause — awaiting approval
$ darktitan ui_
06 · Trust & Control

Autonomous, never unsupervised.

Machine speed, human authority. Built for teams that answer to security reviews and change boards.

Control

A human holds the final say.

Set a gate to escalation: pause and nothing promotes without your sign-off. Stop or steer any run mid-flight.

Security

Guardrails, not guesswork.

Agents run inside the scopes and stop conditions you define. Every plan, change, and deploy is logged.

Scale

Many agents, one console.

Run flows in parallel across projects without losing the thread. The same controls hold at any size.

Enterprise

Against your repositories, under your identity and review process. Your code stays yours.
