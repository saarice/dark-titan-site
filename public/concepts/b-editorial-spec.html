§ 01

The vocabulary of the factory

Six primitives. Learn these and you know how DarkTitan thinks.

noun · 01
Flow
A YAML file defining the full pipeline — a named sequence of stages with prompts, gates, and loops.
noun · 02
Stage
A single unit of work. Each stage runs one Claude Code agent with a specific prompt.
noun · 03
Gate
A quality checkpoint between stages — AI-evaluated, human-approved, or test-passing.
noun · 04
Loop
Automatic retry logic that re-runs a stage, up to a limit, until its gate passes.
noun · 05
Agent
A Claude Code subprocess spawned per stage. Runs autonomously inside your codebase.
noun · 06
Project
A named workspace pointing to a local directory. Flows are assigned to projects before running.

§ 03

From intent to production

Three moves. The first two are yours; the third runs without you.

01

Define the outcome

Describe what to build or modernize in a plain TASK.md.

$ vim TASK.md
02

Configure the pipeline

Set the flow — stages, prompts, gates, loops, escalation.

$ darktitan import flow implement.yaml
03

Run the factory

Launch and watch it ship production-grade code.

$ darktitan run implement.yaml my-app
§ 04

Built to ship software that ships

Engineered for serious work — not demos.

01Multi-Stage PipelinesChain agents into named stages — plan, implement, test, review, deploy — each with its own prompt.
02Quality GatesAI-evaluated, human-approved, or test-passing checkpoints decide whether work advances, loops, or escalates.

06Greenfield DeliveryFrom a blank repo to a deployed, tested service — defined once, delivered end to end.
§ 05

Engineered for serious work

It runs in your codebase

DarkTitan spawns Claude Code agents as subprocesses, right where your code lives. One CLI drives the entire lifecycle.

  • init / new / import / run — stand up a project and launch a flow.
  • status / logs <flow-id> — see exactly what every agent did.
  • pause / resume / stop — take the wheel at any moment.
  • ui — open the live console to watch the factory work.
$ darktitan init
✓ workspace ready
$ darktitan new project my-app
✓ project · ./my-app
$ darktitan import flow implement.yaml my-app
✓ registered · implement-and-review
$ darktitan run implement.yaml my-app
▸ implement … done
▸ review (ai-evaluation) … pass
⏸ pause — awaiting approval
$ darktitan ui
§ 06

Autonomous, never unsupervised

Machine speed, human authority. Built for teams that answer to security reviews and change boards.

Control

A human holds the final say.

Set a gate to escalation: pause and nothing promotes without sign-off. Stop or steer any run mid-flight.

Security

Guardrails, not guesswork.

Agents run inside the scopes and stop conditions you define. Every plan, change, and deploy is logged.

Scale

Many agents, one console.

Run flows in parallel across projects without losing the thread. The same controls hold at any size.

Enterprise

Against your repositories, under your identity and review process. Your code stays yours.
